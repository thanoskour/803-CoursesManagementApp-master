Update Student Details
Data:
  • <student id>
  • <student name>
  • <year of registration>
  • <semester>
Actors
  1. Professor
Pre Conditions
  1. Student item we want to update should exist
Main Flow:
  1. Professor issues "Update Student" command
  2. System prompts professor with form
  3. Professor fills in the data textfields acordingly
  4. Professor presses the 'OK' button
  5. System repeats the 'Browse Student List' use case
Post Conditions
  1. List of students is updated with the new student data
