Add Course Details
Data:
  • <course id>
  • <course name>
  • <syllabus>
  • <year>
  • <semester>
Actors
  1. Professor
Pre Conditions
  1. Professor should have issued the 'Add' command
Main Flow:
  1. Professor fills in the data textfields acordingly
  2. Professor presses the 'OK' button
  3. System repeats the 'Press Browse Button' use case
Field not filled (1)
  1. System notifies professor that a text fields is not filled with data.
  2. System does not proceed and prompts professor to fill in the remaining fields (highlights with red)
Exiting Course (2)
  1. System notifies professor that the course attempting to add exists in the list.
  2. System repeats 'Add Course Details'
Post Conditions
  1. List of courses is updated with the new item
