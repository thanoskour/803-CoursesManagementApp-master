Add Student Details
Data:
  • <student id>
  • <student name>
  • <year or registration>
  • <semester>
Actors
  1. Professor
Pre Conditions
  1. -
Main Flow:
  1. Professor issues the "Add Details" command
  2. Professor fills in the data textfields acordingly
  3. Professor presses the 'OK' button
  4. System repeats the 'Browse Student List' use case
Field not filled (1)
  1. System notifies professor that a text fields is not filled with data.
  2. System does not proceed and prompts professor to fill in the remaining fields (highlights with red)
Exiting Student (2)
  1. System notifies professor that the student attempting to add exists in the list.
  2. System repeats 'Add Student Details'
Post Conditions
  1. List of students is updated with the new item
